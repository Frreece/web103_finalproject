# User Stories

Reference the Writing User Stories final project guide in the course portal for more information about how to complete each of the sections below.

## Outline User Roles

**Manager** Building managers will oversee the buildings. They will have access to viewing and responding to all maintanence requests filled out by tenants.

**Tenant** Tenants are individuals living within a building. Tenants will be able to submit and view their own maintanence requests until they have been taken care of by a Manager.

**Technician** Technicians are maintenance workers who are assigned to requests by managers. They can view their assigned requests and update their assignment status as they begin or complete maintenance work.

## Draft User Stories

1) As a Tenant, I want to submit a maintenance request with a description and photos, so that the manager understands the issue clearly.
2) As a Tenant, I want to view the status of my submitted requests, so that I know whether they've been seen or resolved.
3) As a Tenant, I want to receive a notification when my request status changes, so that I stay informed without having to check manually.
4) As a Tenant, I want to add follow-up comments to an existing request, so that I can provide additional details if needed.
5) As a Manager, I want to view all maintenance requests across my building(s), so that I can prioritize and address them efficiently.
6) As a Manager, I want to filter and sort requests by status, urgency, or date, so that I can quickly identify what needs immediate attention.
7) As a Manager, I want to update the status of a request (e.g., received, in progress, completed), so that tenants are kept informed of progress.
8) As a Manager, I want to mark a request as resolved and leave closing notes, so that there's a clear record of how the issue was fixed.
9) As a Tenant, I want my resolved requests to remain viewable in a history log, so that I can reference past issues if they recur.
10) As a Tenant, I want to select a category of issue, so that the manager knows who to send.
11. As a Manager, I want to assign one or more technicians to a maintenance request, so that the appropriate people are responsible for resolving the issue.
12. As a Technician, I want to view maintenance requests assigned to me, so that I know which issues I need to address.
13. As a Technician, I want to update the assignment status of a request, so that the manager can see whether I have accepted, started, or completed the work.
14. As a Manager, I want the system to prevent incomplete or invalid maintenance requests from being submitted, so that requests contain enough information to be addressed.
